WHAT IS MALWARE?

Malware = Malicious Software — any program intentionally designed to damage systems, steal data, or gain unauthorized access to computing resources.


It covers a broad spectrum: from simple nuisance programs → to sophisticated nation-state tools.

🔑 KEY INSIGHT: Most malware relies on human interaction at some point in the attack chain — making user awareness your first line of defense.

💡 Exam Trick — CIA vs Control
CIA are the classic 3 security pillars. Malware adds a 4th — Control (backdoors, C2). If the exam asks "what does malware target BEYOND the traditional CIA triad?" → Answer: Control / Persistence
📋 ONE-LINE DEFINITIONS TO MEMORIZE:

Confidentiality attack = stealing data nobody should see
Availability attack = making things unavailable (crash/encrypt)
Integrity attack = quietly changing data to wrong values
Control attack = persistent hidden access via backdoor/C2

4 Phases — from spread to defense
🧠 Remember the 4 Phases
"CICP" → Can I Create Problems?

⚡ PHASE 1 — CIRCULATION: HOW MALWARE SPREADS

Every available channel is used to reach victims. Understanding spread = first step to blocking.

📧

EMAIL

Malicious attachments, embedded links, macro-enabled docs — #1 initial access vector

🌐

WEB

Drive-by downloads, malvertising, watering-hole attacks — exploit browsers silently

⬇️

DOWNLOADS

Fake installers, trojanized software, cracked apps — user executes malware voluntarily

🔌

NETWORK

Worms scan for exposed services, exploit unpatched vulnerabilities, spread laterally

💾

REMOVABLE MEDIA

USB / external devices — autorun payloads. Especially dangerous in air-gapped environments

COMMON INFECTION VECTORS (Real World)

📄 Macro-Enabled Docs

Office files with VBA macros — executes when user clicks "Enable Content"

🌐 Browser Exploits

Vulnerabilities in browsers, Java, Flash, PDF readers → silent code execution

🔑 Stolen Credentials

Harvested passwords → attackers log in + manually deploy malware

📦 Compressed Archives

ZIP/RAR hiding executables behind double extensions (e.g., invoice.pdf.exe)

⛓️ Supply-Chain

Compromised software updates distribute malware through trusted vendor channels

🎯 Exam Focus
Email is #1 vector. Supply-chain is most sophisticated. USB is most dangerous in air-gapped environments. Double-extension trick: invoice.pdf.exe — the OS hides .exe so user thinks it's a PDF.

🦠 PHASE 2 — INFECTION: ENTRY & EXECUTION

Once malware reaches a target, it must execute and establish persistence.

1

USER-TRIGGERED EXECUTION

Victim opens a file, runs an installer, or enables macros → initiates the execution chain

Requires human action
2

AUTOMATIC EXECUTION

Exploit chains and worm behavior require NO user interaction — vulnerability alone is enough

Zero user action needed
3

PERSISTENCE MECHANISMS

Ensures malware survives reboots via:

Registry keys Startup folders Scheduled tasks System services
4

PAYLOAD DELIVERY

Final action: steal data, encrypt files, spy on users, control system, or move laterally

👻 PHASE 3 — CONCEALMENT: HOW MALWARE HIDES

Sophisticated malware invests heavily in evasion. Longer undetected = more damage.

1. Obfuscation & Packing

Code is scrambled or compressed so signature-based tools can't match known patterns

2. Anti-VM / Anti-Sandbox

Malware detects analysis environments and refuses to execute, frustrating reverse engineers

3. Encrypted C2 Traffic

Command-and-control disguised as normal HTTPS traffic to blend with legitimate activity

4. Delays & Staged Downloads

Sleeping before execution or downloading payloads in stages to bypass initial scans

5. Rootkits

Kernel-level tools that hide processes, files, and network connections from the OS itself — deepest form of concealment

🎯 Detection vs Evasion = Arms Race
Security = constant tug-of-war. Each side adapts as the other evolves. Rootkits are the hardest to detect because they operate at kernel level — below the OS security layer.

🦠 COMPUTER VIRUSES ▾

A virus attaches itself to a legitimate host file (document, script, or executable) and spreads when that file is opened/executed. Unlike worms, viruses typically require user action to propagate.


Viruses modify host files by inserting their own code. Many carry payloads: data destruction → credential theft.

Program Viruses (Executable Infectors)
Target .exe, .dll, .com files. Every time infected program runs, the virus activates and seeks more executables.

CIH "Chernobyl" (1998) — overwrote flash BIOS on motherboards, physically bricking machines. Most destructive virus ever recorded.

🐛 WORMS ▾

Worms are self-replicating malware that spread across networks WITHOUT needing a host file. They exploit network protocols and vulnerabilities to propagate autonomously, often at devastating speed.

🔁 Self-Replicating

No user action required — copies itself to new hosts automatically

📡 Network Outages

Massive bandwidth consumption + system crashes → widespread disruption

📦 Payload Delivery

Modern worms drop additional malware — ransomware, backdoors, cryptominers

Morris Worm (1988) — First major Internet worm. Infected ~6,000 machines = roughly 10% of the entire Internet at the time.

NotPetya — Used worm-like spread to cripple global enterprises in hours.
🎯 Worm vs Virus — The BIG Difference
Virus = needs a host file + user to open it
Worm = standalone + spreads autonomously via network vulnerabilities
Worms are faster and more dangerous because they don't wait for anyone.
🐴 TROJAN HORSES ▾

Trojans disguise themselves as legitimate, useful software (a game, utility, productivity tool) but carry hidden malicious functionality. Unlike viruses and worms, Trojans do NOT self-replicate — they rely entirely on social engineering to trick users.

🏦 Banking Trojans

Intercept financial transactions, steal banking credentials, inject fake forms into legitimate banking sites

🖥️ RATs

Remote Access Trojans — grant attackers full control: keylogging, screen capture, file access, webcam activation

🔧 Modular Loaders

Emotet-style loaders deliver additional payloads on command — ransomware, info-stealers, lateral movement tools

🎯 Remember
RAT = Remote Access Trojan. It gives the attacker a "remote control" for your entire PC including webcam. Trojans spread via deception, not replication.
💰 RANSOMWARE ▾

Ransomware encrypts victim files or locks entire systems, demanding payment (typically cryptocurrency) for the decryption key. Most financially impactful malware category in the modern threat landscape.

1
INITIAL ACCESS
Phishing email, exposed RDP, or exploited vulnerability
2
PRIVILEGE ESCALATION
Gain admin rights, disable security tools, map the network
3
ENCRYPTION
Mass-encrypt files across drives and network shares
4
RANSOM NOTE
Demand payment with deadline + threat of data publication
⚠️ DOUBLE EXTORTION: Modern ransomware gangs exfiltrate data BEFORE encrypting it, then threaten public release to increase pressure. This means paying the ransom doesn't guarantee your data is safe!

Notable: CryptoLocker (2013) · WannaCry (2017)
🎯 Double Extortion = 2 threats
Threat 1: Files encrypted — can't access your data
Threat 2: Data will be published publicly
Payment solves neither guarantee. Only offline backups solve Threat 1.
🕵️ SPYWARE & KEYLOGGERS ▾
SPYWARE

Silently monitors user activity — browsing habits, app usage, file access, communications. Exfiltrated without victim's knowledge.

  • Screen capture at intervals
  • Clipboard monitoring
  • Microphone/camera activation
KEYLOGGERS

Records every keystroke to capture passwords, credit card numbers, messages. Can be software-based OR physical hardware attached to keyboards.

  • Credential harvesting
  • Session token capture
  • Form-field interception
🛡️ Defense: MFA (Multi-Factor Authentication) · Least-privilege access · EDR (Endpoint Detection & Response) · Regular behavioral monitoring
🎯 Why MFA beats keyloggers
Even if a keylogger steals your password, MFA requires a second factor (phone/hardware token) that the attacker doesn't have. Password alone = useless.
📢 ADWARE & BROWSER HIJACKERS ▾

Adware injects unwanted advertisements (pop-ups, banners, redirected searches) while tracking browsing behavior for profit.

Browser Hijackers go further — change default homepage, search engine, and security settings without consent.

⚠️ Not just a nuisance — real security risks:
• Privacy loss through extensive behavioral tracking
• Exposure to additional malware through malicious ad networks
• Weakened browser security settings → increased attack surface
• Degraded system performance and productivity
🎯 Key Fact
Most frequently bundled with "free" software installers. Always use Custom Installation and review what's included. Adware can serve as a gateway to more serious malware through malicious ad networks.

MODERN THREATS
Zero-day, Botnets, Phishing, WannaCry

🤖 BOTNETS

A botnet is a network of compromised devices (computers, servers, IoT devices) silently controlled by an attacker through a Command-and-Control (C2) infrastructure.

Individual infected devices = "bots" or "zombies"

DDoS ATTACKS

Overwhelm targets with massive traffic volumes, disrupting online services and infrastructure

CRYPTOMINING

Steal compute resources to mine cryptocurrency, running silently in the background

SPAM & PHISHING

Distribute millions of malicious emails from distributed, hard-to-block IP addresses

FRAUD

Credential stuffing, click fraud, and financial theft at scale

📌 MIRAI BOTNET (2016): Exploited default passwords on IoT devices to build a massive botnet. Launched record-breaking DDoS attacks → took down DNS provider Dyn → disrupted Twitter, Netflix, and Reddit.
🎯 Mirai = IoT + Default Passwords = Lesson
Mirai's lesson: Change default passwords on ALL IoT devices. The botnet used the devices' own manufacturer-set passwords. Defense: change defaults, segment IoT networks, update firmware.

🎣 SOCIAL ENGINEERING & PHISHING

Social engineering manipulates human psychology to bypass technical security controls. Phishing is its most common form.

📧 Phishing

Mass emails impersonating trusted brands with generic lures

🎯 Spear Phishing

Highly targeted messages crafted with personal details of specific individuals

📱 Smishing

SMS-based phishing exploiting mobile trust and smaller screen real estate

📞 Vishing

Voice calls impersonating IT support, banks, or government agencies

🧠 Phishing Types Memory Trick
"P-S-S-V" = Phone, SMS, Spear, Voice
Phishing (email) → Spear Phishing (targeted email) → Smishing (SMS) → Vishing (Voice)
🚩 RED FLAGS to spot phishing:
• Urgency language ("Act NOW or your account will be suspended!")
• Unusual sender addresses
• Spoofed domains (paypa1.com vs paypal.com)
• Unexpected attachments
• Requests that bypass normal approval processes
PHISHING DEFENSE STRATEGIES
🎓

TRAINING

Regular phishing simulations + awareness programs so employees recognize and report attacks

📞

VERIFICATION

Out-of-band confirmation for sensitive requests — call back on a known number, don't trust the email

🔐

MFA

Multi-factor authentication ensures stolen passwords alone are NOT enough for account takeover

🔍

FILTERING

Email gateways, URL reputation, attachment sandboxing, DMARC/DKIM/SPF enforcement

🎯 DMARC / DKIM / SPF — What Are These?
SPF (Sender Policy Framework) = specifies which mail servers are allowed to send email for your domain
DKIM (DomainKeys Identified Mail) = cryptographic signature proving email wasn't tampered with
DMARC = policy that tells receivers what to do if SPF/DKIM fail (reject/quarantine)
Together = email authentication trio that blocks spoofed sender addresses

💣 CASE STUDY: WANNACRY (2017)

RANSOMWARE + WORM

WannaCry combined ransomware encryption with worm-like self-propagation — making it uniquely devastating.

  • Exploited EternalBlue — an NSA exploit leaked by Shadow Brokers
  • Targeted unpatched Windows SMB vulnerability
  • Spread to 150+ countries, 200,000+ machines in 24 hours
  • Crippled UK's NHS — hospitals turned away patients
  • Attributed to North Korea's Lazarus Group
🛑 Kill Switch: Security researcher Marcus Hutchins discovered WannaCry checked a specific unregistered domain. He registered it for $10.69 — and the malware stopped spreading globally.
⚠️ Lesson: WannaCry could have been stopped by a patch released 2 months earlier (MS17-010). Unpatched systems = open invitation.
05
DEFENSE PLAYBOOK
Layered controls, patching, backups, incident response
🧠 Defense Layers = "DEPTH"
Defense-in-Depth: No single control is sufficient
Training · Technology · Process · Policy · Human factors — all required together

🔄 PATCHING

Apply security patches promptly. WannaCry's patch (MS17-010) was available 2 months before the attack. Unpatched = open door.

Rule: Patch Critical vulnerabilities within 24-48 hours. High within 7 days. Medium within 30 days.

💾 BACKUPS

The only guaranteed defense against ransomware. Follow the 3-2-1 rule:

  • 3 copies of data
  • 2 different storage media
  • 1 copy offline/offsite (air-gapped)

🔐 MFA & LEAST PRIVILEGE

MFA: Stolen passwords alone can't grant access. Second factor = phone/hardware token.

Least Privilege: Users/programs get only the minimum access they need. Limits blast radius of compromise.

🤖 EDR & ANTIVIRUS

Antivirus: Signature-based detection of known malware.

EDR (Endpoint Detection & Response): Behavioral monitoring — detects anomalies even for unknown malware. More powerful than traditional AV.

🎓 USER TRAINING

Since most malware relies on human interaction at some point — training is your first line of defense.

  • Phishing simulation exercises
  • Recognize red flags
  • Report suspicious activity

🚨 INCIDENT RESPONSE

When compromise happens:

1
Identify — detect and confirm the incident
2
Contain — isolate affected systems
3
Eradicate — remove the malware
4
Recover — restore from clean backups
5
Learn — post-mortem to prevent recurrence
🛡️ KEY PRINCIPLE — DEFENSE IN DEPTH:
No single control is sufficient. Combating threats requires:
Technology (EDR, firewalls, email gateways) + Process (patching, backups, IR plans) + Human Factors (training, awareness, reporting culture)
